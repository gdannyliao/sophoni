# 桌面端 AI Agent 设计规格

日期：2026-06-13

## 1. 背景与目标

本项目要构建一个桌面端为主的 AI Agent。第一版聚焦 macOS 上的本地工作区任务执行，支持多轮会话、文件搜索、文件读取、文件修改、低风险命令自动执行，以及高风险命令拦截或确认。

产品定位是“工作区任务导向的桌面 Agent”，而不是普通聊天应用。底层文件工具尽量保持通用，第一版验收场景以代码项目为主：用户打开一个工作区，向 Agent 描述任务，Agent 搜索和读取相关文件，自动修改文件，展示 diff，必要时运行低风险检查命令，并在会话里总结结果。

第一版不实现移动端和云同步，但架构要为后续移动端接入本机会话预留边界。后续路线是先做同局域网本机遥控，再演进到云同步。

## 2. 范围

### 2.1 MVP 范围

- macOS 桌面应用。
- Tauri + Svelte + TypeScript + Vite 前端。
- Rust/Tauri 后端承载本地 Core Runtime。
- GLM/智谱作为第一版唯一模型 Provider。
- 多轮会话和任务执行记录。
- 文件名、路径、全文关键词搜索。
- 代码符号、Markdown 标题等轻量结构索引。
- 工作区内文件读写自动执行。
- 写文件前快照，写文件后生成 diff。
- 低风险命令自动执行，高风险命令拦截或确认。
- SQLite 本地存储会话、消息、工具日志、索引元数据和变更记录。
- macOS Keychain 保存 API Key。

### 2.2 非 MVP 范围

- 不做完整 IDE 或代码编辑器。
- 不做向量语义搜索。
- 不做多 Provider 实现，只保留接口扩展能力。
- 不做移动端应用。
- 不做云同步。
- 不做插件系统。
- 不做多 Agent 并行执行。

## 3. 总体架构

系统分为三层：

1. Core Runtime
   - Agent 执行循环。
   - 模型 Provider 抽象和 GLM Provider。
   - 文件工具、搜索工具、结构索引工具。
   - 命令执行和风险分类。
   - SQLite repository。
   - 文件快照、diff 和撤销能力。

2. Desktop Shell
   - Tauri + Svelte + TypeScript + Vite。
   - 三栏工作台 UI。
   - 通过 Tauri command/event 调用 Core Runtime。
   - 接收流式模型事件、工具事件、任务状态事件。

3. Access Layer 预留
   - MVP 不开放网络访问。
   - Core Runtime 的接口和事件模型按未来本机安全网关设计。
   - 后续移动端可通过配对授权连接桌面端，查看本机会话、继续发送消息、查看任务进度和处理高风险确认。

Core Runtime 不依赖具体 UI 组件。桌面 UI 是 Core Runtime 的第一个客户端，后续移动端或本地 API 网关可以复用同一套会话、任务、工具和事件模型。

## 4. 桌面技术栈

桌面端采用 Tauri + Svelte + TypeScript + Vite。

选择 Svelte 的原因：

- Svelte 是编译型框架，运行时负担较小，适合长期运行、频繁更新的桌面工作台。
- 会话流、工具状态、diff 面板和文件树会有高频局部更新，Svelte 的响应式模型更轻。
- 相比 React，Svelte 在性能和代码量上更符合本项目偏好。
- 相比 SolidJS，Svelte 生态和学习成本更平衡。
- Tauri 支持任意可编译为 HTML、CSS、JavaScript 的前端框架，后续 Windows/Linux 桌面端不需要重写前端。

跨端策略：

- Windows/Linux 桌面端后续沿用同一套 Svelte 前端和大部分 Rust Core Runtime，只做平台适配。
- 移动端不直接复刻三栏桌面 UI，而是设计更适合手机的远程会话控制台。
- 移动端可复用协议类型、配对逻辑、部分业务逻辑、部分样式 token，但 UI 单独设计。

## 5. Agent 执行与工具系统

第一版采用单 Agent 顺序执行。

用户在会话里输入任务后，Agent Runtime 将以下内容发给 GLM：

- 当前会话历史摘要。
- 当前工作区摘要。
- 相关文件上下文。
- 可用工具 schema。
- 模型配置。

模型可以返回普通文本，也可以发起工具调用。Runtime 执行工具后，把工具结果作为新消息回传模型。循环直到模型给出最终答复，或达到最大步数、最大时间、用户取消、模型错误等停止条件。

### 5.1 工具分类

文件探索工具：

- 列目录。
- 按路径读文件。
- 文件名和路径搜索。
- 全文关键词搜索。
- 结构索引搜索。

文件修改工具：

- 新增文件。
- 修改文件。
- 局部替换。
- 应用 patch。

命令工具：

- 在工作区内执行命令。
- 对命令进行风险分类。
- 低风险命令自动执行。
- 高风险命令拦截或确认。

### 5.2 工具执行规则

- 搜索、读取普通工作区文件自动执行。
- 修改或新增工作区文件自动执行。
- 删除文件、工作区外访问、大规模覆盖默认高风险。
- 低风险命令自动执行，例如 `rg`、`ls`、`git status`、`git diff`、项目测试命令。
- 高风险命令拦截或确认，例如删除/移动大量文件、安装依赖、联网下载脚本、修改系统目录、访问工作区外路径。
- 每次工具调用记录工具名、参数、开始/结束时间、状态、输出摘要和错误。
- 每次文件写入前记录快照，写入后记录 diff。

## 6. 模型 Provider 与 GLM 接入

模型层使用统一的 `ModelProvider` 抽象。Agent Runtime 只依赖内部消息、工具调用和流式事件，不直接依赖具体模型供应商。

MVP 只实现 GLM Provider：

- 支持智谱 API Key 配置。
- API Key 存 macOS Keychain。
- 支持流式文本输出。
- 支持 Function Calling，将本地工具 schema 传给模型，并解析 `tool_calls`。
- 支持 thinking 配置。
- 记录模型名、请求耗时、token 使用量、错误信息。

Provider 接口要预留：

- OpenAI Provider。
- Anthropic Provider。
- Gemini Provider。
- OpenAI-compatible Provider。

模型配置页按多 Provider 结构设计，但第一版只暴露 GLM 配置。其他 Provider 可以隐藏，避免用户误以为已经可用。

## 7. 数据存储与会话模型

第一版使用本地 SQLite 作为主存储。所有数据默认留在用户机器上。API Key 等敏感数据存入 macOS Keychain，SQLite 只保存非敏感配置和引用。

核心实体：

- Workspace：工作区路径、名称、最近打开时间、索引状态。
- Conversation：会话标题、所属工作区、创建/更新时间、当前模型配置。
- Message：用户消息、助手消息、工具调用消息、工具结果消息，支持流式增量落库。
- TaskRun：一次 Agent 任务的状态、开始/结束时间、最终结果、失败原因。
- ToolCall：工具名、参数、风险等级、执行状态、输出摘要、耗时、错误。
- FileSnapshot：写入前文件内容引用或快照哈希。
- FileChange：新增、修改、删除记录、diff、关联工具调用。
- RiskDecision：高风险命令确认、越权访问确认、大规模修改确认记录。
- IndexEntry：文件路径、mtime、大小、语言类型、符号/标题摘要、索引版本。

任务状态至少包括：

- pending
- running
- waiting_for_risk_decision
- completed
- failed
- interrupted
- cancelled

会话恢复规则：

- 用户重开应用后能看到历史会话、任务状态、工具日志和 diff。
- 应用关闭或模型请求失败时，任务标记为 interrupted 或 failed。
- 中断任务不自动继续，用户可基于历史上下文重新发起。

## 8. 搜索与索引

搜索层采用关键词搜索 + 轻量结构索引。

关键词搜索：

- 文件名搜索。
- 路径搜索。
- 全文搜索。
- 代码项目优先使用 `rg` 或等价的本地搜索能力。

结构索引：

- 代码文件提取函数、类、导入关系等符号。
- Markdown 提取标题层级。
- 普通文本提取基础元信息。
- 索引条目包含文件路径、mtime、大小、语言类型、摘要和索引版本。

索引更新：

- 打开工作区时检查索引状态。
- 文件变化后增量更新相关索引条目。
- Agent 写文件后立即更新受影响条目。

MVP 不做 embedding 和向量库，但数据模型保留后续扩展空间。

## 9. 桌面 UI 与交互

主界面采用三栏工作台。

左栏：

- 工作区选择。
- 最近工作区。
- 当前工作区会话列表。
- 模型状态。
- 设置入口。
- 高风险确认提示。

中栏：

- 当前工作区和会话标题。
- 用户消息。
- 助手回复。
- Agent 步骤。
- 工具调用摘要。
- 流式输出。
- 底部任务输入框。
- 任务结束摘要。

右栏：

- 文件树。
- 搜索结果。
- 当前任务相关文件。
- diff 查看器。
- 工具日志。
- 高风险命令确认。
- 快照撤销入口。

关键交互：

- 写文件自动执行，右栏实时显示 diff。
- 低风险命令自动执行，结果进入工具日志。
- 高风险命令被拦截或要求确认。
- 每个工具调用可以展开查看参数、输出摘要和错误。
- 每个任务完成后提供“查看全部改动”和“撤销本任务改动”。
- 设置页包含 GLM API Key、默认模型、thinking 开关、命令风险规则、工作区安全规则。

## 10. 命令风险策略

命令执行采用分级自动执行。

低风险命令：

- 只读或检查类命令。
- 不修改工作区外文件。
- 不联网下载或执行远程脚本。
- 不安装依赖。
- 不启动长期后台进程。

示例：

- `rg`
- `ls`
- `git status`
- `git diff`
- 已配置为低风险的项目测试命令，例如 `npm test`、`pnpm test`、`cargo test`

高风险命令：

- 删除、移动或批量覆盖文件。
- 安装依赖。
- 联网下载脚本并执行。
- 访问或修改工作区外路径。
- 修改系统目录。
- 启动长期后台进程。
- 修改 git 历史或执行破坏性 git 操作。

每次命令执行都记录：

- 命令。
- cwd。
- 风险等级。
- 开始/结束时间。
- 退出码。
- stdout/stderr 摘要。
- 是否被拦截、自动放行或用户确认。

## 11. 错误处理

模型错误：

- API Key 无效。
- 额度不足或限流。
- 模型不支持工具调用。
- 网络失败。
- 响应格式不符合预期。

工具错误：

- 文件不存在。
- 权限不足。
- 文件过大。
- 编码不支持。
- patch 冲突。
- 索引不可用。

命令错误：

- 风险规则拦截。
- 超时。
- 退出码非 0。
- stdout/stderr 过大。
- 工作目录不存在。

所有错误都应作为结构化事件记录到 SQLite，并以可理解方式显示在会话或工具日志中。Agent 可以基于工具错误继续解释、修复或向用户请求下一步。

## 12. 安全策略

- 默认只允许访问当前工作区。
- 搜索、读取、写入工作区文件自动执行。
- 写文件前快照，写文件后 diff。
- 支持撤销单次工具调用或整个任务的文件变更。
- 高风险命令拦截或确认。
- 工作区外路径访问默认阻断。
- API Key 存 macOS Keychain。
- SQLite 不保存明文密钥。
- 工具输出中避免完整记录疑似密钥内容。
- 任务有最大工具步数、最大命令运行时间、最大读取文件大小。

## 13. 测试策略

Rust/Tauri 后端：

- GLM Provider 请求/响应转换测试。
- 工具调用解析和 Agent 循环测试。
- 文件读写权限测试。
- 文件快照和撤销测试。
- SQLite repository 测试。
- 命令风险分类测试。
- 任务中断和失败恢复测试。

Svelte 前端：

- 三栏布局测试。
- 会话流渲染测试。
- 流式输出测试。
- 工具调用展开/折叠测试。
- diff 面板测试。
- 高风险命令拦截/确认 UI 测试。
- 设置页表单测试。

端到端：

- 打开示例代码工作区。
- 用户发起任务。
- Agent 搜索文件。
- Agent 修改文件。
- UI 展示 diff。
- Agent 自动运行低风险测试命令。
- Agent 总结结果。
- 用户撤销任务变更。

回归重点：

- 文件修改不能丢数据。
- 高风险命令不能绕过风险规则。
- 任务失败时日志必须完整。
- 中断任务不能悄悄继续执行。

## 14. 后续路线

阶段 1：macOS MVP

- 完成桌面端、GLM Provider、单 Agent、文件工具、结构索引、SQLite、三栏工作台。

阶段 2：Windows/Linux 桌面端

- 复用 Svelte 前端和 Rust Core Runtime。
- 适配路径、shell、系统密钥存储、打包安装、WebView 差异。

阶段 3：移动端本机遥控

- 桌面端提供本机安全网关。
- 手机端通过配对授权连接桌面端。
- 移动端可查看会话、继续发送消息、查看任务进度、处理高风险确认。
- 桌面端仍负责真实文件操作和 Agent 执行。

阶段 4：云同步

- 增加账户体系和云端同步。
- 同步会话摘要、消息、任务状态和必要元数据。
- 不默认上传本地文件内容，除非用户明确授权。

## 15. 已确认决策

- 产品定位：工作区任务导向桌面 Agent。
- MVP 验收：代码项目优先，底层文件能力通用。
- 平台：macOS 优先。
- 技术栈：Tauri + Svelte + TypeScript + Vite。
- 后端：Rust/Tauri Core Runtime。
- 模型：MVP 只实现 GLM Provider，多 Provider 架构预留。
- Agent：单 Agent 顺序执行。
- 存储：本地 SQLite。
- 密钥：macOS Keychain。
- 搜索：关键词搜索 + 轻量结构索引。
- 文件修改：工作区内自动写入，写前快照，写后 diff。
- 命令：分级自动执行，低风险自动，高风险拦截或确认。
- UI：三栏工作台。
- 移动端路线：第一阶段本机遥控，后期云同步。
